Estándares web

Los estándares web surgen ante la necesidad de un desarrollo web organizado e integrado, anteriormente, las aplicaciones web se comportaban diferente de acuerdo con cada dispositivo o navegador utilizado, cada aplicación requería varias configuraciones para garantizar la compatibilidad en la mayoría de los navegadores.

Los estándares web más representativos son:

HTML

HyperText Markup Language

CSS

Cascading Style Sheets

Javascript

HyperText Markup Language

W3C

World Wide Web Consortium

Comunidad internacional compuesta por organizaciones que estudian, desarrollan, y mantienen los estándares web.

Su objetivo es que los estándares sean de utilidad para reducir costos en tiempo y complejidad

HTML

Lenguaje de etiquetas de hipertexto y el componente mas básico de la web

HTML 5 publicada en el año 2011 por la W3c, fue creada para resolver incompatibilidades que presentaba su versión anterior, no depende de APIs ni de terceros.

Cuerpo de elemento HTML

Etiqueta de apertura: Comienza el elemento y donde se ingresan los diferentes atributos

Etiqueta de cierre: Donde termina el elemento

Tomado de: https://developer.mozilla.org/es/docs/Learn/Getting_started_with_the_web/HTML_basics

Elementos sin etiqueta de cierre algunos elementos no cuentan con etiqueta de cierre, se consideran vacíos, como la etiqueta "img"

Tomado de: https://developer.mozilla.org/es/docs/Web/HTML/Element/img

CSS

Hojas de estilo de cascada

Creado en 1996, es de vital importancia para la correcta visualización y comportamiento entre los componentes HTML.

Estructura de las hojas de estilo

Selector: Definen que elementos se van a aplicar las reglas CSS

Propiedades: Definen las reglas CSS, como los colores, fuente de texto, tamaños de los contenedores, etc.

Valor: El valor es separado de su propiedad por dos puntos, y este indicador depende siempre de la propiedad elegida.

Tomado de: Durango, A. (2015). Diseño Web con CSS: 2ª Edición. IT Campus

JAVASCRIPT

Lenguaje de programación interpretado

Utilizado principalmente para brindarle dinamismo a las aplicaciones web, fue creado en los años 90 por Brendan Eich y actualmente es el lenguaje más usado en tecnologías front-end y es popular en tecnologías back-end.

Tomado de: https://www.freecodecamp.org/news/the-relationship-between-html-css-and-javascript-explained-by-building-a-city-a73a69c6343/

ECMAScript es el estándar creado para Javascript, su primera edición fue en 1997, pero el cambio más importante fue en el 2015, estableciendo una serie de cambios que lo transformarían en un lenguaje moderno. Fueron introducidos aspectos importantes como: Clases, módulos, hashmaps, sets, for of, proxies, etc.

Sintaxis en Js

  • El interprete no procesa los espacios en blanco
  • El interprete diferencia entre mayúsculas y minúsculas
  • Débilmente tipado (no requiere la definición del tipo de variable)
  • No requiere punto y coma al terminar una sentencia
